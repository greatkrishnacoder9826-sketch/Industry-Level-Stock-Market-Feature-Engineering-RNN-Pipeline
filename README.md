A robust, industry-grade data preprocessing and feature engineering pipeline designed for **Deep Learning (RNN / LSTM / GRU)** stock price prediction models. 

This repository demonstrates how to transform raw OHLCV (Open, High, Low, Close, Volume) stock data into powerful sequential indicators using technical analysis, normalize the features, and structure them into time-series sequences for neural networks.

---

## 🚀 Key Features & Technical Indicators

Instead of relying solely on raw prices, this pipeline enriches the dataset with **16 financial features** to capture market trends, momentum, and volatility:

* **Exponential Moving Averages (EMA):**
    * `EMA_10` & `EMA_20`: Captures short-term and medium-term price trends by giving more weight to recent prices.
* **Momentum Indicators:**
    * `RSI (Relative Strength Index)`: Identifies overbought (>70) and oversold (<30) market conditions.
    * `MACD (Moving Average Convergence Divergence)`: Measures trend strength and direction using MACD line, Signal line, and Histogram.
* **Volatility Indicators:**
    * `Bollinger Bands`: Calculates Upper, Lower, and Middle bands to identify periods of high/low volatility and potential price reversals.
    * `Rolling Volatility`: 20-day standard deviation of daily returns to measure market stability.
* **Financial Metrics:**
    * `Daily Returns`: Percentage change in closing price day-over-day.

## ⚙️ How the Pipeline Works

### 1. Data Ingestion & Preprocessing
Loads raw historical stock data (`stock_data.csv`) and computes 11 advanced technical indicators. Initial rows containing `NaN` values generated by rolling windows are cleanly dropped.

### 2. Feature Selection
Selects the 16 most impactful features for sequential modeling:
`Open`, `High`, `Low`, `Close`, `Volume`, `EMA_10`, `EMA_20`, `RSI`, `MACD`, `MACD_Signal`, `MACD_Histogram`, `BB_High`, `BB_Low`, `BB_Middle`, `Daily_Return`, and `Volatility`.

### 3. Data Normalization
Uses `MinMaxScaler` to scale all 16 features into a `[0, 1]` range. This step is crucial for Deep Learning models like RNNs and LSTMs to ensure fast convergence and prevent gradient exploding/vanishing issues.

### 4. Time-Series Sequence Generation
Transforms tabular data into 3D sequential arrays required for RNN architectures:
* **Window Size (`sequence_length`):** `60` timesteps (e.g., looking at the past 60 days of data to predict the next day).
* **Target Variable (`y`):** The next day's `Close` price.
* **Output Shape:** `(Samples, Sequence_Length, Features)` -> `(N, 60, 16)`.

### 5. Train-Test Split
Splits the generated sequences chronologically without data leakage:
* **Training Set:** 80% of historical sequences.
* **Testing Set:** 20% of recent sequences for out-of-sample evaluation.
